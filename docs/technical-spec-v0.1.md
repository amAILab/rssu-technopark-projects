# Техническое задание v0.1

## Сайт управления проектами Технопарка РГСУ

### 1. Цель проекта

Создать рабочий веб-инструмент для руководителя Технопарка РГСУ и участников НТС — научно-технического совета.

Сайт должен использоваться как операционный экран для еженедельных совещаний:

- смотреть состояние проектов;
- видеть человеческие ресурсы и человеко-часы;
- фиксировать вопросы, предложения и решения;
- вести повестку НТС;
- запускать голосования через QR-код;
- учитывать гранты и заявки;
- открывать паспорта проектов;
- отправлять данные в Google Таблицы и агенту Telegram/OpenClaw;
- формировать основу протокола НТС.

Основной сценарий: руководитель Технопарка открывает сайт на совещании НТС и использует его как единый интерфейс управления обсуждением.

### 2. Общая структура

Навигация:

1. Главная
2. Ресурсы
3. Гранты
4. Проекты
5. НТС
6. Сквозной нижний чат

Дизайн: простой, стильный, минималистичный, без перегруза, с крупными цифрами, карточками, фильтрами и удобством работы на экране совещания.

### 3. База данных и синхронизация

Основной источник данных — Google Таблица.

Сайт должен:

- читать данные из Google Таблицы;
- отправлять новые данные в Google Таблицу;
- обновлять статистику;
- передавать события агенту Telegram/OpenClaw.

Рекомендуемый MVP-механизм:

```text
Frontend → Google Apps Script API → Google Sheets → Telegram/OpenClaw агент
```

Fallback: все события обязательно сохраняются в Google Таблицу. Если агент недоступен, событие сохраняется в `AgentQueue`.

- статусы вопросов;
- кнопки запуска голосования;
- QR-код для голосования;

Повестка формируется из:

- вопросов, добавленных руководителем;
- вопросов из формы на главной;
- вопросов по проектам;
- вопросов по грантам;
- вопросов по ресурсам.

Для каждого пункта повестки:

- тема;
- тип вопроса;
- связанный проект;
- докладчик;
- время на обсуждение;
- нужно ли голосование;
- статус;
- решение;
- ответственный;
- срок исполнения.

#### Голосование через QR-код

Сценарий:

1. На экране НТС выбран вопрос.
2. Руководитель нажимает «Запустить голосование».
3. Сайт создаёт голосование.
4. На экране появляется QR-код.
5. Участники сканируют QR-код с телефона.
6. Открывается мобильная форма голосования.
7. Участник голосует.
8. Результаты сохраняются в Google Таблицу.
9. Агент получает событие.
10. На экране отображаются результаты.

Типы голосования:

- за / против / воздержался;
- выбор варианта из нескольких;
- оценка по шкале;
- рекомендовать / не рекомендовать;
- отправить на доработку;
- принять к реализации.

Результаты голосования:

- количество проголосовавших;
- процент «за»;
- процент «против»;
- количество воздержавшихся;
- итоговое решение;
- список участников — только в админском режиме;
- публично — только агрегированные результаты.

### 9. Сквозной чат

Внизу каждой страницы расположен чат-блок в стиле ChatGPT:

```text
[Введите комментарий / поручение / вопрос...] [Отправить]
```

Сообщение сохраняется с контекстом:

- дата и время;
- страница;
- пользователь;
- текст;
- связанный проект / грант / пункт НТС, если есть;
- статус обработки агентом.

Каждое сообщение должно:

- записываться в Google Таблицу;
- отправляться агенту OpenClaw / Telegram;
- попадать в очередь событий;
- использоваться для формирования протокола.

### 11. Интеграция с Telegram/OpenClaw агентом

Агент должен получать события:

- новые вопросы;
- новые заявки от участников;

```json
  "author": {

  }

```

### 16. Критерии приёмки

Сайт считается готовым в MVP, если:

- пользователь может заполнить форму на главной, и данные появляются в Google Таблице;
- доступные, занятые и свободные человеко-часы считаются автоматически;
- на странице ресурсов видно людей, категории, компетенции и загрузку;
- на странице проектов работают поиск, фильтры и переключение вида;
- по клику на проект открывается паспорт проекта;
- в паспорте есть цель, задачи, ответственный, даты и ссылка на Google Drive;
- на странице грантов есть воронка заявок и временная шкала;
- на странице НТС можно создать пункт повестки;
- по пункту НТС можно запустить голосование;
- QR-код открывает мобильную форму голосования;
- голоса сохраняются в Google Таблицу;
- результаты голосования отображаются на экране;
- нижний чат есть на каждой странице;
- сообщения из чата сохраняются с контекстом страницы;
- события отправляются агенту или сохраняются в очередь `AgentQueue`;
- при временной недоступности агента данные не теряются.

- фиксация решений.

#### Этап 7. Чат и агент

- нижний чат на всех страницах;
- сохранение заметок;
- отправка событий агенту;
- очередь необработанных событий;
- подготовка данных для протокола.

### 18. Короткая формулировка задачи для разработчика

Нужно переделать текущий сайт «Проекты Технопарка РГСУ» в рабочий dashboard для управления проектами, ресурсами, грантами и НТС.

Основная база данных — Google Таблица.

Сайт должен позволять участникам оставлять вопросы, регистрировать доступное время, смотреть проекты, открывать паспорта, учитывать человеко-часы, вести грантовую воронку, формировать повестку НТС, запускать QR-голосования и отправлять события агенту Telegram/OpenClaw.

Дизайн сохранить минималистичным, удобным для показа на совещаниях.

### 19. Главное решение по концепции

Сайт должен стать не просто страницей с проектами, а операционной системой НТС Технопарка:

```text
Проекты + люди + часы + гранты + вопросы + голосования + решения + агент + Google Таблица
```

Следующий шаг — зафиксировать структуру Google Таблицы и после этого переходить к переделке сайта по этому ТЗ.
