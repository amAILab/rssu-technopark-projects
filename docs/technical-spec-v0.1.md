# Техническое задание v0.1

## Сайт управления проектами Технопарка РГСУ

### 1. Цель проекта

Создать рабочий веб-инструмент для руководителя Технопарка РГСУ и участников НТС — научно-технического совета.

Сайт должен использоваться как операционный экран для еженедельных совещаний:

- смотреть состояние проектов;
- видеть человеческие ресурсы и человеко-часы;
- фиксировать вопросы, предложения и решения;
- вести повестку НТС;
- запускать голосования через QR-код;
- учитывать гранты и заявки;
- открывать паспорта проектов;
- отправлять данные в Google Таблицы и агенту Telegram/OpenClaw;
- формировать основу протокола НТС.

Основной сценарий: руководитель Технопарка открывает сайт на совещании НТС и использует его как единый интерфейс управления обсуждением.

### 2. Общая структура

Навигация:

1. Главная
2. Ресурсы
3. Гранты
4. Проекты
5. НТС
6. Сквозной нижний чат

Дизайн: простой, стильный, минималистичный, без перегруза, с крупными цифрами, карточками, фильтрами и удобством работы на экране совещания.

### 3. База данных и синхронизация

Основной источник данных — Google Таблица.

Сайт должен:

- читать данные из Google Таблицы;
- отправлять новые данные в Google Таблицу;
- обновлять статистику;
- передавать события агенту Telegram/OpenClaw.

Рекомендуемый MVP-механизм:

```text
Frontend → Google Apps Script API → Google Sheets → Telegram/OpenClaw агент
```

Fallback: все события обязательно сохраняются в Google Таблицу. Если агент недоступен, событие сохраняется в `AgentQueue`.

### 4. Главная страница

Содержит:

- заголовок «Проекты Технопарка РГСУ»;
- универсальную форму;
- быструю статистику;
- кнопку перехода в Telegram-бота;
- краткий блок ближайшего НТС.

Типы обращения:

- вопрос по проекту;
- пожелание или предложение;
- регистрация себя как ресурса;
- доступное время на неделю;
- вопрос для НТС;
- участие в проекте;
- иное обращение.

Обязательные поля:

- ФИО / имя;
- категория участника;
- e-mail;
- Telegram-ник;
- тип обращения;
- текст обращения.

Дополнительные поля:

- проект;
- лаборатория;
- доступные часы;
- дата и время доступности;
- компетенции;
- желаемая роль;
- ссылка на материалы;
- нужно ли вынести вопрос на НТС.

Категории:

- технопарковцы;
- студенты;
- сотрудники университета;
- партнёры;
- волонтёры;
- внешние эксперты.

Минимальная статистика:

- всего проектов;
- активные проекты;
- проекты, требующие решения НТС;
- доступно человеко-часов на неделе.

### 5. Страница «Ресурсы»

Назначение: показывать, кто доступен для работы над проектами, сколько часов у каждого есть, сколько уже занято и сколько осталось. Это инструмент распределения людей и времени.

#### 5.1. Сводка по ресурсам

Карточки:

- всего зарегистрировано участников;
- всего доступно часов на неделю;
- занято часов;
- свободно часов;
- количество студентов;
- количество сотрудников;
- количество партнёров;
- количество волонтёров.

Формула:

```text
Свободные часы = Доступные часы - Распределённые часы
```

#### 5.2. Таблица / карточки людей

Для каждого человека показывать:

- ФИО;
- категория;
- Telegram;
- e-mail;
- компетенции;
- доступно часов на неделю;
- занято часов;
- свободно часов;
- проекты, куда он уже назначен;
- комментарий;
- статус: свободен / частично занят / полностью занят.

#### 5.3. Фильтры

- категория участника;
- компетенции;
- лаборатория;
- свободные часы больше 0;
- занятость;
- проект;
- неделя.

#### 5.4. Учёт человеко-часов

Нужна отдельная таблица назначений:

```text
человек → проект → неделя → количество часов → задача/роль
```

Поля: неделя, человек, проект, роль, назначено часов.

### 6. Страница «Гранты»

Назначение: управление грантами, заявками и грантовой воронкой.

#### 6.1. Воронка грантов

Этапы:

- всего потенциальных проектов;
- отобрано под гранты;
- заявка готовится;
- заявка написана;
- заявка подана;
- прошла формальную проверку;
- получена поддержка / выигран грант;
- отклонено / требуется доработка.

#### 6.2. Диаграмма Ганта по грантам

Для каждого гранта:

- название;
- фонд / конкурс;
- дедлайн;
- этап подготовки;
- ответственный;
- связанные проекты;
- статус;
- ссылка на документы;
- краткое описание;
- ключевые даты.

Этапы: анализ конкурса, сбор данных, подготовка заявки, согласование, подача, ожидание результата, отчётность.

#### 6.3. Карточка гранта

По клику на грант открывается карточка:

- название гранта;
- фонд / программа;
- сумма;
- дедлайн;
- статус;
- ответственный;
- связанные проекты;
- описание;
- что нужно подготовить;
- список документов;
- ссылка на Google Drive;
- история решений по гранту.

### 7. Страница «Проекты»

Назначение: просмотр, поиск, фильтрация и открытие паспортов проектов.

#### 7.1. Лаборатории

Фильтрация по трём лабораториям:

- Промдизайн и инжиниринг;
- Компьютерная наука и искусственный интеллект;
- Робототехника и БПЛА.

#### 7.2. Представления

- плитки — визуальный просмотр;
- таблица — совещание, сортировка, сравнение;
- минимальный список — быстрый поиск.

#### 7.3. Фильтры проектов

- лаборатория;
- статус;
- ответственный;
- наличие гранта;
- требуется решение НТС;
- наличие паспорта;
- приоритет;
- свободный поиск по названию, описанию, ответственному.

#### 7.4. Паспорт проекта

Паспорт содержит:

- название проекта;
- лаборатория;
- статус;
- цель;
- задачи;
- краткое описание;
- ответственный;
- команда;
- партнёры;
- ближайшие даты;
- текущие проблемы;
- решения НТС;
- необходимые ресурсы;
- уже выделенные человеко-часы;
- связь с грантами;
- ссылка на Google Drive;
- история обновлений.

#### 7.5. Индикатор заполненности паспорта

Для каждого проекта отображается процент заполненности паспорта. Индикатор нужен для быстрого понимания, какие проекты готовы к обсуждению и какие требуют дооформления.
