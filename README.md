# Проекты Технопарка РГСУ

MVP веб-инструмента для управления проектами Технопарка РГСУ и проведения НТС.

## Цель

Сайт используется не как витрина, а как операционный экран для еженедельных совещаний НТС:

- быстрый статус проектов;
- доступные ресурсы и человеко-часы;
- вопросы, предложения и решения;
- повестка НТС;
- голосования через QR-код;
- гранты и заявки;
- паспорта проектов;
- отправка данных в Google Sheets и Telegram/OpenClaw агента;
- основа для протокола НТС.

## Навигация

- Главная
- Ресурсы
- Гранты
- Проекты
- НТС
- Сквозной нижний чат на каждой странице

## Архитектура MVP

```text
Frontend site → Google Apps Script API → Google Sheets → Telegram/OpenClaw agent
                                         ↘ AgentQueue fallback
```

Основной источник данных — Google Таблица. Все формы, голосования, комментарии, решения и изменения ресурсов должны сохраняться в таблицу.

Если локальный агент недоступен, событие не теряется: оно пишется в лист `AgentQueue`, откуда агент сможет забрать его позже.

## Запуск локально

```bash
npm install
npm run dev
```

## Переменные окружения

Создайте `.env` на основе `.env.example`:

```bash
VITE_SHEETS_API_URL=https://script.google.com/macros/s/DEPLOYMENT_ID/exec
VITE_TELEGRAM_BOT_URL=https://t.me/your_bot
```

## Короткая задача для разработчика

Переделать сайт «Проекты Технопарка РГСУ» в рабочий dashboard для управления проектами, ресурсами, грантами и НТС.

Ключевая концепция:

```text
Проекты + люди + часы + гранты + вопросы + голосования + решения + агент + Google Таблица
```

Основная база данных — Google Таблица. MVP должен писать данные через Google Apps Script, сохранять события в `AgentQueue` и не терять данные при недоступности локального Telegram/OpenClaw агента.

## Ближайший следующий шаг

1. Утвердить структуру Google Таблицы из `docs/sheets-schema.md`.
2. Создать таблицу и листы.
3. Вставить `apps-script/Code.gs` в Google Apps Script.
4. Указать `SPREADSHEET_ID` и URL backend в `.env`.
